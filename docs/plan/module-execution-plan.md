# 模块执行计划

## 🎯 总体执行策略
应用核心转型为：**每日引导练习 (Guided Daily Practice)** + **专项自由练习 (Targeted Practice)**。

### 开发原则
1. **引导优先**: 优先保障 1-N 的线性学习路径闭环。
2. **专项模块化**: 每个内容模块独立可接入。

## 📅 每日引导路径 (Guided Daily Practice)
- [ ] **路径引擎**: 实现线性任务调度系统
- [ ] **进度追踪**: 持久化用户当日学习位置 (SQLite)
- [ ] **交互流**: 引导用户按预设顺序完成当日核心任务

## 📚 内容专项练习模块

### 1. 基础内容层 (音标、字词根、单词)
- [ ] **音标系统**: 交互式发音纠正
- [ ] **字词根系统**: 构词法拆解与练习
- [ ] **单词系统**: 记忆卡片、测试模式、遗忘曲线复习

### 2. 进阶内容层 (句子、文章、听力)
- [ ] **句子系统**: 语法拆解、排序、翻译
- [ ] **文章阅读**: 长难句精读、理解力训练
- [ ] **听力系统**: 场景化听力训练、听写练习

## ⚙️ 技术实现细节

### 数据库适配 (SQLite)
- **PathTable**: 存储路径节点关系
- **UserProgressTable**: 记录用户在特定路径的当前 N 值
- **ModuleContentTable**: 各模块具体练习内容

### 状态管理
- 使用 **Provider/Riverpod** 统一管理当前学习状态 (Daily Path vs. Free Mode)

## 🚀 执行时间表 (第一阶段)

### 第1-2周: 路径引擎与框架重构
- 路径状态机设计
- 专项练习模块的抽象基类定义

### 第3-4周: 核心专项练习填充
- 单词与字词根功能实现
- 句子与阅读模块数据接入

### 第5-6周: 听力练习与系统完善
- 音频处理与听力题目实现
- 每日练习数据统计可视化
