# 英语学习应用 - V1 MVP 版本需求文档

## 1. 版本目标与范围

V1 版本定位为应用的 **MVP (最小可行性产品)** 阶段。本版本的主要目标是搭建起应用的核心框架，打通“每日练习”的引导流程，并实现基本的本地数据持久化和用户系统底座。本阶段数据以本地 Mock 数据和 SQLite 为主，暂不涉及网络请求和云端同步。

---

## 2. 核心功能点定义

### 2.1 每日学习引导路径
*   **路径跳转引擎 (Linear Scheduler)**:
    *   以“单词卡片 $\rightarrow$ 单词测试 $\rightarrow$ 句子练习 $\rightarrow$ 文章精读”为标准线性步骤。
    *   页面支持点击“下一步”进行无缝衔接。
*   **打卡完成状态**:
    *   当日四大步骤全部完成后，在首页触发“今日已完成打卡”的祝贺动效，并更新本地连续打卡天数。

### 2.2 基础学习模块
*   **单词卡片与测试**:
    *   展示单词拼写、发音喇叭（本地 TTS 或 Mock 播放）、音标、释义、中英双语例句。
    *   提供中译英/英译中单项选择题测验。
*   **句子三合一练习**:
    *   **排序模式**: 拖拽或点击碎片重组句子。
    *   **填空模式**: 挖空助动词/介词，用户填入正确词汇。
    *   **翻译模式**: 中英双语对照翻译的选择题。
*   **文章详情精读**:
    *   文章按句子拆分展示，支持点击任意句子高亮。
    *   中英双语对照翻译一键开关。
    *   字体大小动态调节（12px 至 24px 缩放）。

### 2.3 本地数据存储与持久化 (SQLite + SharedPrefs)
*   **SharedPreferences**:
    *   用户设置保存（深色/浅色模式、字体大小、通知开关）。
    *   用户学习记录（打卡天数、今日背词个数、今日学习分钟数）。
*   **SQLite**:
    *   **生词本**: 存储用户在背单词或阅读中手动加入的生词。
    *   **错题集**: 记录练习中做错的单词和句子。

### 2.4 个人中心与成就系统
*   **统计展示**: 累计学习时长、累计单词数、累计文章数。
*   **本地成就柜**: 预设 6 个基础成就（如：连续打卡 7 天、背词 500 个等），自动读取本地打卡和背词数据来点亮或置灰。
*   **学习目标设定**: 提供每日背词 10/20/30 个等选项，用户修改后保存至 SharedPreferences 并改变首页进度条总量。

## 3. 视觉规范实现说明 (V1 MVP)
*   基本适配 [Master PRD](./master_prd.md) 中定义的 **温润极简 (Warm Minimalism)** 设计风格：
    *   使用自定义 widget [modern_ui.dart](../../lib/widgets/modern_ui.dart) 渲染扁平淡雅卡片、绿色渐变按钮与统计圆环。
    *   支持流畅的主题切换（默认浅色明亮模式，支持切换深色模式）。
